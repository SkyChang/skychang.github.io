VSO - 使用Release Management 搭配 Azure 和 Visual Studio Online 佈署篇

發表於 2015-02-11   |  

前言

前面我們存檔完成後，其實就可以看到，剛剛的範本已經準備好了，我們只需要按下”新增發行”，就可以觸發發行的動作。

開始發行

接下來，於跳出的視窗下，我們選擇最新的組件，並且啟動。
過程可能會很久，請耐心的等待…. ( 至少20~30min )。

過了很長一段時間，就可以看到畫面顯示已經完成了。

那如果之前有設定要簽核，則可以從我的簽核要求這邊看到請求簽核；因為這邊的範例，只有一台Dev的機器，所以按下核准後，也不會進入到QA的機器，但正式環境底下，Dev的主管，在測試完畢之後，按下確定，就會開始佈署QA的環境。

按下簽核後，會跳出一個視窗，讓簽核者，能輸入一些訊息。

基本上，整個Release Management的設定到佈署就到這邊告一個段落。
